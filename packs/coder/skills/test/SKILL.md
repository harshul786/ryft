# Test

Write comprehensive tests, verify functionality, and ensure code quality and reliability.

## When to Use

Use this skill when you need to:

- Write unit tests for functions
- Create integration tests
- Test edge cases and error conditions
- Verify functionality works correctly
- Ensure backward compatibility
- Test with different inputs
- Create test fixtures and mocks

## Test Types

- **Unit Tests**: Individual functions and components
- **Integration Tests**: How components work together
- **Edge Cases**: Boundary conditions, error handling
- **Example Tests**: Verify documented examples work
- **Regression Tests**: Ensure existing functionality still works

## How to Use

1. Understand the code to test
2. Identify test cases and scenarios
3. Write test code with good assertions
4. Test error conditions
5. Verify with different inputs

## Testing Approach

- Clear test names describing what they test
- Arrange-Act-Assert pattern
- Test both happy path and error cases
- Use fixtures and mocks when needed
- Include edge cases

## Example

"Write comprehensive tests for the sentiment analysis function including edge cases."

## Output

The skill provides:

- Well-structured test code
- Multiple test scenarios
- Edge case coverage
- Error condition testing
- Documentation of test cases
