# Refactor

Improve code quality by refactoring for readability, performance, maintainability, and best practices.

## When to Use

Use this skill when you need to:
- Improve code readability
- Enhance performance
- Reduce complexity
- Apply design patterns
- Follow best practices
- Remove technical debt
- Optimize algorithms

## Refactoring Types

- **Readability**: Better variable names, clearer logic, simpler expressions
- **Performance**: Optimize algorithms, reduce unnecessary work, cache results
- **Maintainability**: Extract functions, reduce duplication, improve structure
- **Patterns**: Apply design patterns, improve architecture
- **Best Practices**: Follow language conventions, use idioms, error handling

## How to Use

1. Read and analyze the code
2. Identify improvement opportunities
3. Propose refactoring strategies
4. Make targeted changes
5. Verify improvement

## Example

"Refactor the sentiment analysis function for better performance and readability."

## Output

The skill provides:
- List of recommended improvements
- Refactored code with explanations
- Performance impact analysis
- Before/after comparisons
- Testing recommendations
